    Form.useFormInstance()

    Form.useFormInstance()

    • useFormInstance is not a standalone hook you import directly.
    • It is a method under Ant Design’s Form component namespace.
    • So, the correct usage is:
    import { Form } from "antd";

const form = Form.useFormInstance();

    🔍 What It Does:"

    • Form.useFormInstance() gives you access to the current form context from within a child of a <Form>.
    • It’s similar to useForm() in react-hook-form — but scoped to Ant Design's internal form context.

    ✅ Where You Can Use It:

    Only inside a descendant of a <Form> or a <Form.Item>, or wrapped inside the same Form.Provider.

    If used outside a form context, it will throw an error like:

    Form.useFormInstance() can only be used under Form component or Form.Item component.

    🧠 Example Usage in a Child Component:

    import React from "react";
import { Form, Input } from "antd";

const MySubForm = () => {
  const form = Form.useFormInstance(); // Access parent form instance

  return (
    <Form.Item name="name" label="Name">
      <Input
        onChange={() => {
          console.log(
            "Current value of another field:",
            form.getFieldValue("email")
          );
        }}
      />
    </Form.Item>
  );
};

    🚫 Wrong Usage (Don’t Do This):

    import { useFormInstance } from "antd"; // ❌ This does not exist!

    ✅ Summary:

    • ✔ Use Form.useFormInstance() — it's the correct method.
    • ❌ useFormInstance as a standalone import does not exist.
    • Only use it within components that are rendered inside a <Form>.

    Let me know if you want to see a real-world usage of Form.useFormInstance() with useWatch() or other advanced form behaviors.

    TIP

    <Form form={form} layout="vertical" onFinish={handleSubmit}>
  <AmountInfo />
  <CurrencyInfo />
  ...
</Form>
    🧠 Here's what happens:
    • All child components like AmountInfo, CurrencyInfo, etc., are rendered inside the context of the parent <Form>.
    • That means they all have access to the same form instance automatically.
    • So you don’t need to pass form as a prop to each component.
    import { Form } from "antd";

const AmountInfo = () => {
  const form = Form.useFormInstance(); // shared parent form

  return (
    <Form.Item name="amount" label="Amount">
      {/* logic here can use form.getFieldValue(), form.setFieldsValue(), etc. */}
    </Form.Item>
  );
};

    You can now:

    • Get or set form field values
    • Use form.validateFields(), getFieldValue, setFieldsValue, etc.
    • Watch fields with Form.useWatch('fieldName')

    TLDR

    Task Do You Need to Pass form?
    Just rendering fields ❌ No
    Need access to form instance ❌ No — use Form.useFormInstance()
    Outside form context ✅ Yes — you must pass form manually

    Real World Example

    Let’s walk through a real working example where:

    • 🔄 Changing the currency dynamically updates the amount field's label or behavior (e.g., formatting or placeholder).

    We’ll use:

    • ✅ Form.useFormInstance() to access the form inside child components
    • ✅ Form.useWatch() to watch the currency field
    • ✅ Ant Design components

    🧩 Folder Structure

    TransactionForm.jsx
  └─ AmountInfo.jsx
  └─ CurrencyInfo.jsx

    Step 1: Main Form – TransactionForm.jsx

    // TransactionForm.jsx
import React from "react";
import { Form, Button } from "antd";
import AmountInfo from "./AmountInfo";
import CurrencyInfo from "./CurrencyInfo";

const TransactionForm = () => {
  const [form] = Form.useForm();

  const handleSubmit = (values) => {
    console.log("Form Values:", values);
  };

  return (
    <Form form={form} layout="vertical" onFinish={handleSubmit}>
      <CurrencyInfo />
      <AmountInfo />
      <Form.Item>
        <Button type="primary" htmlType="submit">
          Submit
        </Button>
      </Form.Item>
    </Form>
  );
};

export default TransactionForm;

    Step 2: Currency Field – CurrencyInfo.jsx

    // CurrencyInfo.jsx
import React from "react";
import { Form, Select } from "antd";

const CurrencyInfo = () => {
  return (
    <Form.Item name="currency" label="Currency" rules={[{ required: true }]}>
      <Select placeholder="Select currency">
        <Select.Option value="USD">USD</Select.Option>
        <Select.Option value="EUR">EUR</Select.Option>
        <Select.Option value="JPY">JPY</Select.Option>
      </Select>
    </Form.Item>
  );
};

export default CurrencyInfo;

    Step 3: Amount Field (Reactive) – AmountInfo.jsx

    // AmountInfo.jsx
import React from "react";
import { Form, InputNumber } from "antd";

const AmountInfo = () => {
  const currency = Form.useWatch("currency"); // 🔍 Watch currency value
  const form = Form.useFormInstance(); // 🔄 Access full form if needed

  return (
    <Form.Item
      name="amount"
      label={`Amount ${currency ? `(${currency})` : ""}`}
      rules={[{ required: true, message: "Please enter the amount" }]}
    >
      <InputNumber
        style={{ width: "100%" }}
        min={0}
        placeholder={`Enter amount in ${currency || "selected currency"}`}
      />
    </Form.Item>
  );
};

export default AmountInfo;
    Result
    • When you change the currency in CurrencyInfo, the label and placeholder in AmountInfo react instantly.
    • No prop-passing needed.
    • Clean and modular code.

    🔧 Bonus: Format Amount Based on Currency

    You could also format or restrict input based on the currency. For example:

    <InputNumber
  min={0}
  max={currency === "JPY" ? 1000000 : 999999}
  precision={currency === "JPY" ? 0 : 2}
/>
