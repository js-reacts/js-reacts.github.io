    Overview

    Rule of Thumb

    • If it needs React (like hooks or lifecycle), use a custom hook.
    • If its just a computation or decision, use a pure function.

    Real-Life Example

    • business/forexLogic.js (Pure business logic)
    export function getResetFields(type) {
  return type === "BUY"
    ? { sellCurrency: undefined }
    : { buyCurrency: undefined };
}
    • hooks/useResetOnTypeChange.js (UI + lifecycle logic)
    import { useEffect } from "react";
import { getResetFields } from "../business/forexLogic";

export function useResetOnTypeChange({ type, form }) {
  useEffect(() => {
    const resetFields = getResetFields(type);
    form.setFieldsValue(resetFields);
  }, [type]);
}

    Summary

    Task Type Put in Hook? Example
    React lifecycle, useEffect Yes useFormSync
    Local component state abstraction Yes useTableFilter
    Conditional logic or rules No getResetFields()
    Computations, data transformations No calculateRateDiff()

    THEORY

    Standard Pattern to Separate Business and UI Logic in React

    1. Component (UI) Layer

    • Purpose: Handles UI rendering and UI-specific behavior (event handlers, form updates, etc.)
    • Contains: useState, useEffect, JSX, component-level logic

    Example:

    src/components/ForexForm.js

    2. Hooks Layer (Optional but Recommended)

    • Purpose: Contains reusable, component-level logic (e.g., form handling, data fetching).
    • Contains: React logic that isnt UI, but still uses React (like useEffect or useState)
    • File prefix: use*.js

    Example:

    src/hooks/useForexDealLogic.js

    3. Business Logic Layer

    • Purpose: Pure JS logic no React or UI dependencies. Easy to test.
    • Contains: Decisions, calculations, transformations based on business rules

    Example:

    src/business/forexHelpers.js or forexLogic.js

    4. Services Layer (API / backend interaction)

    • Purpose: Abstract API calls and communication with backend services.
    • Contains: fetch, axios, or any data access logic

    Example:

    src/services/forexService.js

    5. Constants & Types

    • Central place for enums, transaction types, field names, etc.

    Example:

    src/constants/forex.js

    import { FOREX_DEAL, TRANSACTION_TYPE } from "../constants/forex";

export function getFormFieldsToReset(transactionType) {
  const baseFields = {
    [FOREX_DEAL.SELLING_AMOUNT]: undefined,
    [FOREX_DEAL.BUYING_AMOUNT]: undefined,
    [FOREX_DEAL.DEAL_APPROVERS]: null,
    [FOREX_DEAL.DEAL_VERIFIERS]: null,
  };

  if (transactionType === TRANSACTION_TYPE.BUY) {
    baseFields[FOREX_DEAL.SELLING_CURRENCY] = undefined;
  } else if (transactionType === TRANSACTION_TYPE.SELL) {
    baseFields[FOREX_DEAL.BUYING_CURRENCY] = undefined;
  }

  return baseFields;
}

    Custom Hooks

    Understanding when to use hooks and how they relate to business logic vs UI logic is key to writing scalable React apps.

    When to Create Custom Hooks

    Custom hooks are useful when you want to encapsulate and reuse stateful logic or side effects. That logic may involve:

    • React state or effects (useState, useEffect, etc.)
    • DOM or event handling
    • Form state handling
    • API calls
    • Lifecycle events tied to UI or state

    Hooks Are for React Logic Not Pure Business Logic

    Logic Type Use a Custom Hook? Why?
    UI-related logic (state, effects, etc.) Yes Hooks encapsulate React-specific logic
    Business rules (calculations, decisions) No These should go into pure functions, not hooks
    Combined logic (e.g. data fetch + UI reset) Yes Wrap business + UI logic if side effects are needed

    Breakdown: UI Logic vs Business Logic in Hooks

    Use Hooks For UI Logic When:
    Situation Example
    Side effects need to be reused useFormResetOnTypeChange
    Local state reused across components usePagination, useModal
    React logic is repeated across files useApi, useFetch, useDebounce
    Youre abstracting interactions with libraries useAntForm, useTableSorting
    Do Not Use Hooks For:
    Situation Example
    Pure functions with no useState or useEffect calculateExchangeRate, getResetFields()
    Stateless rules/validation isUserEligible(deal)

    Rule of Thumb

    • If it needs React (like hooks or lifecycle), use a custom hook.
    • If its just a computation or decision, use a pure function.

    1. Create a Business Logic Utility File

    utils/calculateRateChange.js
    export function calculateRateChange(rateAmount, currentRate) {
  const difference = rateAmount - currentRate;
  const percentage = (difference / currentRate) * 100;

  return {
    difference: difference.toFixed(2),
    percentage: percentage.toFixed(2),

/**************************************************************************************/

// utils/calculateRateChange.ts

export interface RateChange {
  percentage: string;
  difference: string;
}

export function calculateRateChange(
  rateAmount: number,
  currentRate: number
): RateChange {
  const difference = rateAmount - currentRate;
  const percentage = (difference / currentRate) * 100;

  return {
    difference: difference.toFixed(2),
    percentage: percentage.toFixed(2),
  };
}

    This is a pure function no React state, no side effects perfect for unit tests or reuse.

    2. Use This Logic in Your Custom Hook

    hooks/useRateCalculation.ts
    import { useEffect, useState } from "react";
import { calculateRateChange } from "../utils/calculateRateChange";

export const useRateCalculation = (
  rateAmount,
  currentRate,
  buyingCurrency,
  sellingCurrency
) => {
  const [rateChange, setRateChange] = useState(null);
  const [toleranceValue, setToleranceValue] = useState({
    min: null,
    max: null,
  });

  useEffect(() => {
    if (!buyingCurrency || !sellingCurrency) {
      setRateChange(null);
      setToleranceValue({ min: null, max: null });
      return;

// hooks/useRateCalculation.ts

import { useEffect, useState } from "react";
import { calculateRateChange, RateChange } from "../utils/calculateRateChange";

interface Tolerance {
  min: number | null;
  max: number | null;
}

export const useRateCalculation = (
  rateAmount: number,
  currentRate: number | null,
  buyingCurrency?: string,
  sellingCurrency?: string
) => {
  const [rateChange, setRateChange] = (useState < RateChange) | (null > null);
  const [toleranceValue, setToleranceValue] =
    useState < Tolerance > { min: null, max: null };

  useEffect(() => {
    if (!buyingCurrency || !sellingCurrency) {
      setRateChange(null);
      setToleranceValue({ min: null, max: null });
      return;
    }

    if (currentRate != null) {
      const result = calculateRateChange(rateAmount, currentRate);
      setRateChange(result);
    }
  }, [rateAmount, currentRate, buyingCurrency, sellingCurrency]);

  return { rateChange, toleranceValue };
};

    MISC with Tolerance calc

    Great let's also separate the tolerance calculation into a reusable function and plug it into your existing hook.

    Step 1: Create a Business Logic Function

    // utils/calculateTolerance.js
export function calculateTolerance(currentRate, tolerancePercent = 5) {
  if (!currentRate || typeof currentRate !== "number")
    return { min: null, max: null };

  const min = currentRate - (currentRate * tolerancePercent) / 100;
  const max = currentRate + (currentRate * tolerancePercent) / 100;

  return {
    min: parseFloat(min.toFixed(4)),
    max: parseFloat(max.toFixed(4)),
  };
}

    You can adjust tolerancePercent as needed (defaults to 5%).

    ** Step 2: Update Your Hook to Use It

    // hooks/useRateCalculation.js

import { useEffect, useState } from "react";
import { calculateRateChange } from "../utils/calculateRateChange";
import { calculateTolerance } from "../utils/calculateTolerance";

export const useRateCalculation = (
  rateAmount,
  currentRate,
  buyingCurrency,
  sellingCurrency
) => {
  const [rateChange, setRateChange] = useState(null);
  const [toleranceValue, setToleranceValue] = useState({
    min: null,
    max: null,
  });

  useEffect(() => {
    if (!buyingCurrency || !sellingCurrency) {
      setRateChange(null);
      setToleranceValue({ min: null, max: null });
      return;
    }

    if (currentRate != null) {
      const result = calculateRateChange(rateAmount, currentRate);
      const tolerance = calculateTolerance(currentRate);

      setRateChange(result);
      setToleranceValue(tolerance);
    }
  }, [rateAmount, currentRate, buyingCurrency, sellingCurrency]);

  return { rateChange, toleranceValue };
};

    Example Output

    If currentRate = 100 and tolerancePercent = 5, then:

    calculateTolerance(100);
// => { min: 95.0000, max: 105.0000 }

    Hooks Testing

    Testing React hooks like useRateCalculation usually involves using React Testing Librarys @testing-library/react-hooks or the built-in renderHook utility.

    Heres a quick step-by-step guide to set it up and test your hook:

    1. Install dependencies

    npm install --save-dev @testing-library/react @testing-library/react-hooks jest

    Note: If you use React 18+, install the compatible version:

    npm install --save-dev @testing-library/react-hooks@next

    2. Create test file for your hook

    features/forex/hooks/__tests__/useRateCalculation.test.js

    import { renderHook } from "@testing-library/react-hooks";
import { useRateCalculation } from "../useRateCalculation";

describe("useRateCalculation hook", () => {
  test("returns null rateChange and toleranceValue when currencies are missing", () => {
    const { result } = renderHook(() =>
      useRateCalculation(100, 100, null, null)
    );

    expect(result.current.rateChange).toBeNull();
    expect(result.current.toleranceValue).toEqual({ min: null, max: null });
  });

  test("calculates rateChange and toleranceValue correctly with valid inputs", () => {
    const { result } = renderHook(() =>
      useRateCalculation(105, 100, "USD", "EUR")
    );

    expect(result.current.rateChange).toEqual({
      difference: "5.00",
      percentage: "5.00",
    });

    expect(result.current.toleranceValue).toEqual({
      min: 95,
      max: 105,
    });
  });

  test("updates values when inputs change", () => {
    const { result, rerender } = renderHook(
      ({ rateAmount, currentRate, buying, selling }) =>
        useRateCalculation(rateAmount, currentRate, buying, selling),
      {
        initialProps: {
          rateAmount: 100,
          currentRate: 100,
          buying: "USD",
          selling: "EUR",
        },
      }
    );

    expect(result.current.rateChange).toEqual({
      difference: "0.00",
      percentage: "0.00",
    });

    rerender({
      rateAmount: 110,
      currentRate: 100,
      buying: "USD",
      selling: "EUR",
    });

    expect(result.current.rateChange).toEqual({
      difference: "10.00",
      percentage: "10.00",
    });
  });
});
